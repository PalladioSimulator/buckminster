The ultimate goal for this description is to install the rest of Buckminster, specifically in order to install Buckminster commands beyond the basic ones. But, to do this, a number of dependencies must be fulfilled first.

The dependencies are on other features (and their plugins). These features can be downloaded from their update sites (or actually, from other ‘things’ that are usable as ‘sites’). Three comments on this at this point:

1)      Some of the features are unfortunately somewhat coarse-grained and there are many plugins installed that you sometimes don’t need – if final size matters greatly, there are advanced ways of pruning the final result.

2)      As we’re presently basing Buckminster on Eclipse 3.2Mn, i.e. a beta, there is no actual update site posted for this. When 3.2 is released the update site will be located at http://update.eclipse.org/updates/3.2. For the moment though we will use a 3.2 Mn Eclipse IDE installation as our resource for 3.2 material – thus, unless you already have an installation of 3.2 Mn please download and unpack.

3)      Also, large parts of the API for interaction with update manager sites is marked as ‘interim’, i.e. made available with the understanding that it will/may change before being final – thus, instructions here can only be said to have been accurate for 3.2 Mn. Also, as the API lacks more in-depth documentation, some things are not completely understood and will be revisited later to improve this prodecure.

 Using the update mechanisms

 There are three commands available: listsite, install and uninstall. Actually, there is also a command (currently hidden) that may be of use during configuration: ‘dumpinfo’. This command is generally for support reasons and will dump various information including features and plugins installed.

Using listsite is very simple – just provide it with a path or URL and it will list any features it finds there. Sample:

buckminster listsite http://download.eclipse.org/technology/buckminster/updates

Searching
http://download.eclipse.org/technology/buckminster/updates...
|**************************************************************| 100,0%      17
Features:
org.eclipse.buckminster.feature_1.0.0.M2sp001 (Buckminster)
org.eclipse.buckminster.headless.feature_1.0.0.M2sp001 (Buckminster Headless)

org.eclipse.buckminster.p4.ui.feature_1.0.0.M2sp001 (Buckminster - Perforce support (UI)) 
org.eclipse.buckminster.svn.feature_1.0.0.M2sp001 (Buckminster - Subversion support) 
org.eclipse.buckminster.ui.feature_1.0.0.M2sp001 (Buckminster (UI))

Note that it works just as well for the product itself:

buckminster.exe listsite c:\buckminster

Searching file:/C:/buckminster/...
|**************************************************************| 100,0%       4
Features: 
org.eclipse.buckminster.headless.feature_1.0.0.M2sp002 (Buckminster Headless) 
org.eclipse.buckminster.headless.platform.feature_1.0.0.M2sp002 (Headless Platform) 
org.eclipse.buckminster.product_1.0.0.M2sp002 (Buckminster Product)

Commands for install/uninstall are very similar: for install, point to a site and a feature you wish to install (and an optional version of it if there are multiple, default is to take the latest). For uninstall, just give the feature name and optional version.

Installing required features

The org.eclipse.platform feature is fairly big but Buckminster is dependent on several of the plugins. Installing it can be done through pointing it at the site which has a 3.2 version of it (i.e. currently only your installed 3.2 Mn Eclipse IDE):

buckminster.exe install c:\etc\eclipse org.eclipse.platform

Searching for org.eclipse.platform_0.0.0 in file:/c:/eclipse/...
|**************************************************************| 100,0%      21
Installing org.eclipse.platform_3.2.0.v20051214-5wrXvetwgj...
|**************************************************************| 100,0%      57

Now we can install the real Buckminster core feature:

buckminster install http://download.eclipse.org/technology/buckminster/updates org.eclipse.buckminster.feature

|**************************************************************| 100,0%      19
